# 📝 Interactive Task Manager

A sleek, functional To-Do List application built with HTML5, CSS3, and Vanilla JavaScript. This project focuses on DOM manipulation, local storage persistence, and dynamic UI filtering.
## 🧠 Lessons Learnt

Building this application provided deep insights into the core pillars of front-end development:

### 1. The Power of the DOM
I learned how to use `document.createElement` and `appendChild` to modify a webpage in real-time without needing a full page reload. Understanding the difference between `innerText` and `innerHTML` was crucial for security and performance.

### 2. Event Delegation
Instead of adding an event listener to every single task (which is inefficient), I learned to use **Event Delegation**. By listening for clicks on the parent `<ul>`, I can catch events from any child `<li>` or `<span>`, even those added *after* the page has loaded.

### 3. State Management with LocalStorage
Learning how to turn JavaScript objects into strings and back again to save data locally was a "lightbulb moment." It taught me how to make a web app feel like a real desktop application.

## ⭐ My Experience as a Beginner Developer

As a beginner, this project was both challenging and rewarding. At first, CSS positioning and JavaScript logic felt like separate worlds. However, connecting them—making a button click change a CSS class which then triggers a visual transition—felt like magic. 

The biggest takeaway was the importance of **debugging**. Watching the console and tracing why a function didn't run taught me to think more like a programmer: step-by-step, logical, and patient.
